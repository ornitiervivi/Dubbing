# Subagents
Quando usar subagents: tarefas complexas multi-domínio.
Subagents: Harness Architect, Clean Architecture Guardian, API/Application Engineer, Persistence/Storage Engineer, AI Provider Integration Engineer, Security/Abuse Prevention Engineer, DevOps/Observability Engineer, Testing/Evals Engineer.
Responsabilidades: cada subagent declara escopo, arquivos, limites, conflitos e entrega.
Evitar conflitos: ownership por pasta, integração pelo agente principal.
Relatório final obrigatório por subagent.
