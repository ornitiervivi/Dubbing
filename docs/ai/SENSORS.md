# Sensors
Sensores mandatórios: git status --short, git diff --name-only, git diff --numstat, mvn test, mvn verify, docker compose config, docker build -t cloud-dubber-api:local .
Sensores de qualidade: ArchUnit tests, Testcontainers integration tests, code review rubric, smell scan manual por subagent, dependency boundary review, API contract review, security review, no-binary check, no-secret check, no-Kotlin check, no-record check, no-code-comments check.
ArchUnit deve garantir: domain sem Spring/JPA/Jackson/HTTP; presentation não acessa repository direto e chama apenas application/use case; infrastructure não é usada pelo domain; DTO HTTP e JPA entity não entram no domain; providers externos ficam em infrastructure; config em infrastructure/config; use case não depende de controller nem de JPA concreto.
