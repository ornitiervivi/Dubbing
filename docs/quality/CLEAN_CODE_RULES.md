# Clean Code Rules
Classes com responsabilidade única, métodos com uma ação clara, nomes intencionais de classes/métodos/variáveis de domínio. Proibido usar nomes vagos como Manager, Helper, Util, Processor ou Handler genérico sem contexto. Evitar muitos parâmetros e boolean flags; usar command/request/criteria object quando necessário. Evitar if/else profundo, duplicação, primitive obsession e stringly typed domain; criar value objects quando houver ganho de clareza/segurança. Não retornar null quando houver alternativa melhor. Usar StringUtils.hasText, Objects.nonNull/isNull/equals e CollectionUtils.isEmpty/isNotEmpty quando fizer sentido. Não usar comentários para explicar código ruim. Exceções devem ser claras. Validações no nível correto. Não misturar parsing, validação, regra de negócio e persistência. Não misturar HTTP com application/domain nem provider externo com regra de negócio.
