# Engineering Principles
Todo código deste projeto deve seguir obrigatoriamente os princípios: Clean Architecture, SOLID, Clean Code, domain-driven thinking quando fizer sentido, Hexagonal Architecture (Ports and Adapters), alta coesão, baixo acoplamento, encapsulamento, imutabilidade quando fizer sentido, composition over inheritance, dependency inversion, boundaries explícitos, unidades pequenas de comportamento, testability by design, fail fast, secure by default, observable by default, simple before clever, YAGNI, KISS, DRY com bom senso, Tell Don’t Ask quando fizer sentido, Law of Demeter quando fizer sentido, command/query separation quando fizer sentido, idempotência para operações de pipeline quando aplicável, transaction boundaries explícitas, consistent error handling, sem framework leakage no domínio, sem infrastructure leakage em application/domain, sem fake abstractions e sem nomes genéricos que escondam design ruim.
